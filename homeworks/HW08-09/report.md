# HW08-09 – PyTorch MLP: регуляризация и оптимизация обучения

## 1. Кратко: что сделано

- **Датасет:** EMNIST (balanced) – вариант B. 47 классов (цифры и буквы), 28×28 пикселей, grayscale. Выбран как компромисс между скоростью обучения на CPU и содержательностью задачи.
- **Часть A (регуляризация):** Сравнение моделей: Base (без регуляризации), Dropout (p=0.3), BatchNorm, и лучшей модели с Early Stopping.
- **Часть B (оптимизация):** Диагностика learning rate (слишком большой/маленький) и сравнение Adam с SGD+momentum+weight decay.

## 2. Среда и воспроизводимость

- **Python:** 3.10+
- **torch / torchvision:** 2.10.0+cpu / 0.25.0+cpu
- **Устройство:** CPU (Intel Core i7-12700H)
- **Seed:** 42 (torch, numpy, random)
- **Как запустить:** открыть `HW08-09.ipynb` и выполнить Run All.

## 3. Данные

- **Датасет:** EMNIST Balanced
- **Разделение:** train_full (112800) → random_split (80/20) → train_ds (90240), val_ds (22560), test_ds (18800) из torchvision
- **Трансформации:** ToTensor() + Normalize((0.5,), (0.5,)) → значения в [-1, 1]
- **Комментарий:** 47 классов, 28x28 grayscale. Задача средней сложности, хорошо подходит для демонстрации эффектов регуляризации.

## 4. Базовая модель и обучение

- **Модель MLP:** 2 скрытых слоя (256 → 128), активация ReLU, вход 784 (28×28), выход 47 классов.
- **Loss:** CrossEntropyLoss
- **Базовый Optimizer (часть A):** Adam (lr=0.001)
- **Batch size:** 64
- **Epochs (макс):** 20
- **EarlyStopping:** patience=5, min_delta=0.001, метрика val_accuracy

## 5. Часть A (S08): регуляризация (E1-E4)

- **E1 (base):** 2 скрытых слоя 256→128, без Dropout и BatchNorm
- **E2 (Dropout):** как E1 + Dropout(p=0.3) после каждого скрытого слоя
- **E3 (BatchNorm):** как E1 + BatchNorm1d после каждого линейного слоя (до активации)
- **E4 (EarlyStopping):** лучший из (E2/E3) – BatchNorm – обучен с EarlyStopping (patience=5)

## 6. Часть B (S09): LR, оптимизаторы, weight decay (O1-O3)

- **O1:** Adam, lr=0.1 (слишком большой), 5 эпох
- **O2:** Adam, lr=1e-5 (слишком маленький), 5 эпох
- **O3:** SGD, lr=0.01, momentum=0.9, weight_decay=1e-4, 8 эпох

- **O3 (SGD+momentum+wd):** Стабильное обучение, результат близкий к Adam (0.8498 vs 0.8524)

## 8. Анализ

- **Переобучение:** В E1 (base) виден разрыв между train и val accuracy (~5%), train loss продолжает падать, val перестаёт расти после 11 эпохи.  
  **Dropout (E2)** уменьшил переобучение (разрыв сократился до ~2-3%), но немного снизил абсолютную точку.  
  **BatchNorm (E3)** не только ускорил сходимость (выход на плато к 5-6 эпохе), но и дал лучший результат, при этом разрыв минимальный.

- **EarlyStopping (E4):** Остановил обучение на 17 эпохе (patience=5), сохранил лучшие веса. Результат практически идентичен полному обучению (0.8523 vs 0.8524), сэкономлено 3 эпохи.

- **O1 и O2:**  
  - LR=0.1: loss скачет, нестабилен, accuracy не растёт выше 0.77 – оптимизация "перепрыгивает" минимум.  
  - LR=1e-5: loss снижается крайне медленно, за 5 эпох accuracy едва достигла 0.67 – шаг слишком мал.

- **O3 (SGD+momentum+wd):** Показал стабильное обучение, уступив Adam всего ~0.3%. Weight decay (L2-регуляризация) помогает контролировать веса, momentum сглаживает градиенты. Для данного датасета Adam всё же эффективнее.

- **Почему BatchNorm лучший:** EMNIST содержит 47 классов с вариативностью написания. BatchNorm нормализует активации, ускоряет сходимость и стабилизирует обучение, что критично для MLP на такой размерности.

## 9. Итоговый вывод

- **Базовый конфиг:** MLP с 2 скрытыми слоями (256→128), BatchNorm, Adam (lr=0.001). Это даёт лучшее соотношение скорость/качество.
- **Что улучшить:** Попробовать более глубокую архитектуру (3-4 слоя) с Dropout для борьбы с переобучением.
